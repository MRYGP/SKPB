# 破壁 · PB

「破壁」是一个假设侧研究仓库：用第一性原理找出各行业**早该被拔掉、却还没人拔的神圣假设**，并照亮由此打开的颠覆式位置。

它不是产品库，也不是落地执行库。它只做判断：照亮可能出现颠覆式机会的位置；落地、商业化、现金流判断一律交给 SK。

## 核心问题

> 这个行业所有人都默认、却从没被质疑的那个前提是什么？  
> 如果把它拔掉，会重定义出什么问题？

当前阶段的愿景棱镜是「AI 交付结果」：

以前是“我们有强大的 AI，你来学会用它”；终局是“你有一个任务，AI 已经帮你做完了，你看一下对不对”。破壁用这个终局倒推：如果 AI 直接交付结果，哪个行业假设会塌？

## 三库星座

| 库 | 角度 | 核心提问 | 结果 |
| --- | --- | --- | --- |
| SK | 供给侧·落地 | 什么变得可能了 | 中国能落地什么菜 |
| LJKL | 需求侧·照亮 | 谁的被锁需求第一次能被服务 | 照亮空格子 |
| 破壁 | 假设侧·照亮 | 哪个神圣假设早该被拔掉 | 照亮颠覆式位置 |

三者刻意错开，互不污染。破壁不下场，输出止于“破坏式滩头第一步 → 交 SK”。

## 方法

破壁使用“马斯克假设爆破四步”：

1. **识别神圣假设**：列出行业默认但不质疑的前提。
2. **第一性拆到原子**：不问市场怎么定价，问原子成本、基本功能和成本黑洞。
3. **重定义问题**：把成本黑洞和被拔掉的假设，翻译成一个新问题。
4. **终局倒推**：推演 3-5 年后假设塌掉后的现实，再倒推今天该瞄准哪里。

每个正式候选必须具备七要素：

1. 拔掉的神圣假设
2. 第一性拆解
3. 重定义的问题
4. 3-5 年终局愿景
5. 同位墓碑
6. 类型判定 + 证据等级
7. 破坏式滩头第一步 → 交 SK

## 铁律

- 分析单元是「神圣假设」，不是产品。
- 无墓碑不入档：每个候选必须说明以前谁试过、为何死、现在窗口变了什么。
- 不功利：不在愿景层引入变现、中国落地、现金流、价值捕获判据。
- 只照亮不落地：不展开 SK 才该做的执行细节。

## 当前产出

- `TASK-001`：电商「AI 交付结果」首爆。
- `TASK-002`：记账「AI 交付结果」第二爆。
- `CAND-001`：电商运营不必由卖家掌握。
- `CAND-002`：记账与申报不必由懂账的人持续操作。

`CAND-001` 的主判断是：电商的成本黑洞不只是工具费用，而是把真实生意连续翻译成平台动作的人类运营层。被照亮的人群是有货、有履约能力、但被现有电商复杂度挡在门外的线下卖家和小卖家。

`CAND-002` 的主判断是：财税记账与申报的成本黑洞不在记账软件，而在把连续经营事实翻译成干净账、报表和申报草稿的月度人力层。被照亮的人群是有真实经营活动、但请不起会计也学不会准则的人。

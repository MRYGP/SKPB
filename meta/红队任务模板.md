# 红队任务模板

## 使用方法

把本模板复制到 ChatGPT 红队 Project，对方已读 `meta/ChatGPT红队指令.md` 后使用。每次只红队一个候选或一条裁决，不要一次塞多个目标。

使用前确认 ChatGPT Project 已开启 web 搜索。若未开启，红队只能提出待查 claim，不能把反证墓碑当事实。

## 模板

```markdown
# 红队任务

> 目标：CAND-___ / TASK-___ 裁决 / 某条具体判断

## 一、当前 Claude 判断（供攻击，不是前提）

请用 3-8 行写清楚 Claude 当前判断：

- 被拔掉的神圣假设：
- Claude 认为它能被拔掉的理由：
- Claude 使用的主墓碑 / 证据：
- Claude 声称「窗口变了什么」：
- 当前裁决或证据等级（如有）：

注意：以上只是攻击靶，不是你必须接受的前提。

## 二、合法击中轴（先读，防保守误杀）

红队只能用以下理由打回或降级：

- Claude 说「窗口变了」，但你证明窗口没真变。
- Claude 说历史死因已消失，但你证明死因今天仍活着。
- Claude 假设某窗口能溶解锁点，但你证明该锁点是不变结构锁（法律责任 / 清算主体 / 信任担保等），任何可预见窗口变化都不触及。
- Claude 引用的墓碑/证据死因不匹配本爆签名。
- Claude 说门槛塌陷，但你证明只是位移，且新门槛仍是结构性锁点。

以下理由无效，不得作为打回依据：

- 当前很难。
- 当前没人成功。
- 当前看起来结构上锁着，但你没有证明它是不被窗口触及的不变结构锁。
- 当前不像生意或不可变现。

这些只是颠覆式候选的常态，不是反证。

打回前必须点明：这是会被窗口变化溶解的现状难度，还是窗口触及不到的不变结构锁。

## 三、红队攻击目标

请只攻击以下问题，不要扩散：

1. Claude 说的窗口是否真变了？若没变，哪条历史死因仍活着，或哪条不变结构锁未被触及？
2. Claude 的主墓碑是否同位？死因是否匹配本爆签名？
3. 是否存在门槛位移：从 A 门槛换成 B 门槛，而不是门槛消失？
4. 是否存在未被质疑的更深元假设？

## 四、必须寻找的反证

请优先找硬反证，不要泛泛谈风险：

- 同位死者：有人试过拔同一假设，失败，且失败条件今天仍在。
- 反向研究：证明 Claude 说「塌陷」的门槛只是位移，或出现沉默失败。
- 签名错配：Claude 引用的证据/墓碑死因不匹配本爆签名。
- 不变结构锁：法律责任、清算主体、信任担保等不被当前窗口触及的永久锁。
- 低估的约束：法规、责任、验收、数据、反作用力等结构性约束。若只是现状难度，不得作为打回依据。

每条反证都必须带来源链接。找不到就写「未找到，暂时扛住了」，不要脑补。

反证墓碑默认是**待查 claim**，只有同时满足「来源可核实 / 案例同位 / 死因匹配 / 窗口未变」四项，才可建议 Claude 据以降级；否则必须标「待 Codex 查实」。

## 五、输出格式

请按以下格式输出：

### 一、最强反对意见

给出这个候选/裁决最可能被打回的结构性理由，但必须落在「窗口未变 / 死因仍在 / 不变结构锁 / 签名错配 / 门槛位移」之一。不得以当前难度作为理由。若主张不变结构锁，必须说明它为何不被 Claude 所说窗口触及。

### 二、反证墓碑

| 反证墓碑 | 谁试过 | 为何死/退守 | 窗口为何没真变 | 来源 | 核实状态 |
|---|---|---|---|---|---|
| | | | | | 已核实 / 待 Codex 查实 / 丢弃 |

### 三、沉默错配

| Claude 说法 | 红队指出的错配 | 实际证明了什么 | 来源 |
|---|---|---|---|
| | | | |

### 四、异议纯度自检

| 异议 | 是否同位/结构性 | 若不纯，是否丢弃 |
|---|---|---|
| | 是 / 否 | 保留 / 丢弃 |

### 五、红队建议

从以下四类中选一类，但只作为建议，不替 Claude 裁决：

- 打回
- 降级
- 加限定条件
- 扛住了

并用 3-6 行解释为什么。

「扛住了」是好结论，不是失败。找不到硬反证时，明确写扛住了。

## 六、越权禁止

- 不写候选。
- 不定证据等级。
- 不下成立结论。
- 不做变现 / 中国落地 / SK 种菜判断。
- 不写仓库文件，不动 git。
```

## 第一发建议

```markdown
目标：TASK-006 裁决。

Claude 当前裁决：
验收门槛未塌，是模式位移，且有沉默地板。Panko 电子表格错误研究可作为验收侧反向同位墓碑：电子表格是「非专家 authoring 一次性合身软件 + 行为自验」的历史自然实验，证明用户会自信地交付有隐藏错误的产物，且发现不了自己的错。因此 CAND-004 可写但封顶 B，CAND-005「人类验收环节不必存在」不开。

你的任务：
1. 攻击这个裁决。
2. 专攻 Panko / spreadsheet 是否真能作为同位墓碑。
3. 查 AI 是否已经改善沉默错误发现率，尤其是非专家行为验收。
4. 找反证墓碑或反例。
5. 不写候选，不定证据等级，不下成立结论，只给红队建议。
```

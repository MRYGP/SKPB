# 维护规约

- 文件名不带版本号，版本写在文件内首行。
- 纲领类文件（`00-宪章.md` / `meta/Project指令.md` / `方法-马斯克假设爆破.md`）改动必须单独 commit，并写明理由。
- 一候选一次 commit，一任务一次 commit。
- 默认 push `origin main`。
- 编辑只改本次触发部分，不做无关整理。
- 拿不准就停下问，不要自圆其说。
- 扫墓碑/查实类工单：Codex 按工单内判据表(rubric)联网填，结局分类与"死因是否匹配签名"为硬项，不匹配须标"不支持签名·勿强塞"，只回贴证据+预判+存疑，不写候选、不定证据等级、不下成立结论（裁决归 Claude）。

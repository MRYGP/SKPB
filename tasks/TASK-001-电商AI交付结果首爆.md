# TASK-001 · 电商「AI 交付结果」首爆

## 任务

对「电商」行业跑一次完整的假设爆破，产出 1 个主候选 + 若干旁候选，存入 `候选档案/`。

## 种子

- 领域：电商
- 愿景棱镜：AI 交付结果（终局＝"你有一个任务，AI 已经帮你做完，你看一下对不对"）

## 执行步骤（按 方法-马斯克假设爆破.md 四步）

1. 列出电商行业所有人默认、却没人质疑的神圣假设（至少 6 条）。
2. 选其中最强的一条，走完四步：识别假设 → 第一性拆到原子（找成本黑洞）→ 重定义问题 → 3-5 年终局倒推。
3. 按七要素立档为 `候选档案/CAND-001-*.md`（缺一不立档）。
4. 其余有潜力的假设，各用一行记为旁候选，附在主候选档案末尾或单独立档。

## 七要素模板（缺一不立档）

1. 拔掉的神圣假设
2. 第一性拆解（原子成本 / 成本黑洞在哪）
3. 重定义的问题
4. 3-5 年终局愿景
5. 同位墓碑（以前谁试过拔这个假设 / 为何死 / 现在窗口变了什么）
6. 类型判定（S032 类型 A 低端破坏 / 类型 B 基础设施重构）+ 证据等级 A/B/C/X
7. 破坏式滩头第一步 → 交 SK（只给指针，不展开落地）

## 验收标准（自检通过才算完成）

- [ ] 主候选七要素齐全，第 5 项有真实同位死者（不是泛指）
- [ ] 全程未套用 SK 落地判据（价值可捕获 / 中国优先 / 现金流）——这些只能出现在第 7 项"交 SK"的指针里，不能用来在前 6 项里筛候选
- [ ] AI 被当作杠杆/棱镜，不是主角（没退化成"AI 能做什么功能"）
- [ ] 至少 3 个旁候选，证明引擎在真生成而非回声
- [ ] **隐藏验证**：独立跑出的主候选，是否落在"线上运营/电商运营被 AI 重构、目标=被现有电商挡在门外的线下卖家"这一带？落在 → 引擎与预演收敛，逻辑稳；明显偏离 → 不是错，但要回报杠，对照预演看是引擎更优还是跑偏。

## 反漂移自检（出候选前过一遍，见 00-宪章）

- 在问"AI 能做什么"？→ 改问"哪个假设能拔"
- 在扫现有玩家做分类？→ 训练集是提取签名用的
- 在愿景层套"能不能赚钱"？→ 那是 SK 滩头判据，挪到这会误杀
- 被既定题目框死？→ 问"谁现在被这个行业完全挡在门外"

# TASK-004 · 软件不必是「持久通用制品」第四爆

## 换沟说明（限制#4 自检）

前三候选签名同构（干掉把真实事实翻译成合规动作的人类/中心中间层：电商运营层 / 记账人力层 / 平台信任层）。本爆刻意换沟：攻击的是「制品必须持久且通用」这条假设，不是中间人。

## 锁定假设

软件必须是「被造出来、然后被学会使用」的持久通用制品——因为为单一任务/单一用户一次性造刚好合身、用完即弃的软件，其制作成本历来高到不可行。

## 第一性拆解（初判，待 CAND 细化）

- 成本黑洞：SaaS 卖的是「通用+持久+需被学会」的坐席/订阅租金；真实原子成本是「解决这一个具体实例」的一次性生成，正趋近零。溢价 = 用户被迫为通用性、持久性、学习曲线付费，尽管他只要这一次的结果。
- 被挡在门外的人：所有有明确一次性/长尾任务、但学不会或不值得学一个通用 SaaS 的人（长尾小企业、非技术岗、临时需求）。

## 重定义的问题

当「为单个任务造一个刚好合身、用完即弃的软件」成本趋近零，软件为什么还要是通用的、持久的、需要被学会的？
